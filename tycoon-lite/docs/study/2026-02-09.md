## 새로 배운 것

### 이벤트 우선순위 (EventPriority)
- `MONITOR`: 가장 마지막에 실행, 다른 플러그인이 취소했는지 확인 후 처리
- `HIGH`, `NORMAL`, `LOW`, `LOWEST`: 순서대로 실행
- `ignoreCancelled = true`: 이미 취소된 이벤트는 무시

### 템플릿 메서드 패턴 (AbstractJobExpService)
- 공통 로직(`addExp()`)은 부모 클래스에 구현
- 직업별 특화 로직(`addMiningExp()`)은 하위 클래스에서 구현
- `addMiningExp()` → `addExp()` 호출로 공통 로직 재사용

### 데이터 저장 패턴
- `markDirty()`: 데이터가 변경되었음을 표시
- 실제 저장은 비동기로 나중에 수행 (성능 최적화)
- `PlayerDataManager`가 변경된 데이터를 주기적으로 저장

### 등급별 보너스 시스템
- `GradeBonusConfig`: 등급별 보너스 설정 관리
- `getXpMultiplier()`: 등급에 따라 경험치 배율 적용
- 순환 의존성 방지를 위해 `Function<UUID, JobGrade>` 형태로 주입
